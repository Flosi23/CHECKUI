CHECKUI uses the copy and paste approach which means that as you add more components you might also have to add more dependencies.
This guide will only help you to set up absolute basic dependencies and files you will always need.

## Prerequisites

Before you can start using CHECKUI, you need to have the following tools installed and files present:
- [tailwindcss](https://tailwindcss.com/), a `tailwind.config.js` file and a global `index.css` file
- [react](https://reactjs.org/)
Without them you won't be able to use CHECKUI.

## Base Setup

<Note>In the following we assume that all [Prerequisites](#prerequisites) are met</Note>

### Install dependencies
Install the `@raix-ui/react-slot` dependency that is used in almost every component.

```bash
npm i @radix-ui/react-slot
```

Install `tw-merge`, `clsx` and `class-variance-authority` to merge tailwind classes and add class variance.
```bash
npm i tw-merge clsx class-variance-authority
```

### Utils file
Create a `utils.ts` file in your project (often referenced as `@lib/utils` by our components) and paste the following content:

```ts
import { type ClassValue, clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

```

```css

@tailwind components;
@tailwind utilities;

    --primary: 206 100% 33%;
    --primary-variant: 205 98% 38%;
    --primary-variant-dim: 205 99% 32%;
    --primary-container: 201 81% 96%;
    --on-primary: 0 0% 100%;
    --surface-primary: 213 90% 24%;
    --surface: 0 0% 100%;
    --surface-dim: 240 19% 95%;
    --surface-container: 0 0% 96%;
    --surface-variant: 0 0% 98%;
    --on-surface: 0 0% 20%;
    --on-surface-light: 0 0% 40%;

  }
}
```
